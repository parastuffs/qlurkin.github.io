C'est lent !!

Performance

Ce n'est pas juste le temps de chargement !

Perception

Comment la rapidité est perçue ?

  • 16 millisecondes: animations fluides (60fps).
  • 0.1 seconde: instantané, sensation de réactivité.
  • 1 seconde: garde l'attention de l'utilisateur.
  • 10 secondes: l'utilisateur est passé à autre chose.

Feedback

Il faut réagir aux inputs de l'utilisateur en moins de 0.1 seconde

Feedback

  • Feedback visuel pour chaque action
  • Indicateur de chargement pour ce qui dure plus longtemps
  • Limiter les long Tasks

Long Tasks

Les long tasks monopolisent le thread principal et retarde les autres tâches.

Temps de chargement

Un chargement ne devrait pas prendre plus d'une seconde

Premier chargement

  • First Paint (FP): Premier rendu par le navigateur (en général très vide)
  • First Contentfull Paint (FCP): Premier rendu avec du contenu
  • First Meaningfull Paint (FMP): Premier rendu avec du contenu utile
  • Time to Interactive (TTI): La page devient interactive

Temps de chargement

  • Limiter la taille des ressources à chargerMoind de code, uglify-js, uglifycss, ...
  • Limiter le nombre de ressources à chargerBundle, inline, ...

Animations

Les animations devraient avoir un framerate constant (et de 60fps si possible)

  • Animations qui utilisent le GPUCSS, WebGL
  • Animations qui demandent moins de calculsCSS transform
  • Eviter les long tasks

Real User Measurments (RUM)

Les mesures durant le développement ne suffissent pas

Feedback: Mesure


				menuButton.addEventListener("click", function (event) {
					if(menuShown) hideMenu()
					else showMenu()

					const lag = performance.now() - event.timeStamp
					if(lag > 100) {
						report('input-latency', {
							event_label: '#menuButton:click',
							value: Math.round(lag)
						})
					}
				})

Long Tasks: Mesures


				if('PerformanceLongTaskTiming' in window) {
					const longTaskObserver = new PerformanceObserver((list) => {
						for (const entry of list.getEntries()) {
							report('longtask', {
								value: Math.round(entry.startTime + entry.duration),
								event_label: JSON.stringify(entry.attribution)
							})
						}
					});

					longTaskObserver.observe({entryTypes: ['longtask']});
				}

Configuration

A mettre au début de la balise <head>


				<!-- Global site tag (gtag.js) - Google Analytics -->
				<script async src="https://www.googletagmanager.com/gtag/js?id=GA_TRACKING_ID"></script>
				<script>
					window.dataLayer = window.dataLayer || [];
					function gtag(){dataLayer.push(arguments);}
					gtag('js', new Date());

					gtag('config', 'GA_TRACKING_ID');
				</script>

Envoyer les Changements de page

Avec une Single Page App, il faut envoyer manuellement les changements de page.


				gtag('config', 'GA_TRACKING_ID', {
					'page_title' : 'homepage',
					'page_path': '/home'
				});

Envoyer des évenements

On peut envoyer des evenements

  • Metriques de performanceslong tasks, lags, tti, ...
  • Convertions
  • ...

				gtag('event', <action>, {
					'event_category': <category>,
					'event_label': <label>,
					'value': <value>
				});

Notre fonction report():


				const report = function (name, attribs) {
					console.log(name, attribs)
					
					attribs.event_category = 'Performance Metrics'
					gtag('event', name, attribs);
				}
