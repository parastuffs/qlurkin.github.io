TW3C - Administration Linux - Séance 1

Linux est un système d'exploitation multi-utilisateurs et multi-tâches. Linux étant un système multi-utilisateurs, il est nécessaire de se connecter sous un nom d'utilisateur au démarrage d'un système Linux.

Le login permet de distinguer les utilisateurs entre eux, ce qui permet à plusieurs personnes de travailler simultanément sur le même système, chacun étant assuré que lui seul est autorisé à accéder à ses fichiers.

Le login est également une sécurité pour le fonctionnement de la machine. Le compte utilisateur permet d'éviter de modifier ou de supprimer par erreur des fichiers nécessaires au fonctionnement du système, auxquels seuls l'utilisateur root (l'administrateur) a accès.

Nom_de_la_machine login:

Introduisez votre nom d'utilisateur suivi de votre mot de passe. Aucun caractère n'apparaît durant l'introduction du mot de passe. Linux est sensible aux majuscules pour tout (nom d'utilisateur, mot de passe, noms de fichiers...)

Le nom d'utilisateur utilisé ici est student et le mot de passe est student.

Une fois loggé apparaît une ligne d'invite

nom_d_utilisateur@nom_de_la_machine:~$

Dans notre cas:

student@tw3c:~$

Cette ligne indique le nom d'utilisateur, le nom de la machine et le nom du répertoire courant. Le ~ désigne votre répertoire de travail personnel, situé sous /home/nom_d_utilisateur.

Le système est en attente de commandes. Le programme actuellement en cours d'exécution est le shell. Ce qui est visible à l'écran est l'invite du shell. Le shell est l'interpréteur de commandes du système. (shell = coquille autour du noyau)

Note: le signe $ représente l'invite du shell

$ exit

exit permet de quitter le système ou de sortir d'un shell.

Consoles virtuelles

Linux étant multi-utilisateur et multitâche, il permet d'exécuter plusieurs programmes en parallèle, et pour des utilisateurs différents. Pour cela, il existe dans la plupart des distributions Linux des consoles virtuelles. Celles-ci sont accessibles depuis le mode texte au moyen de alt-F1 à alt-F6. La console 7 étant réservée à l'affichage du système X-Windows (interface graphique).

Passez en revue les différentes consoles avec alt-f1 à alt-f7 ou ctrl-alt-F1 à ctrl-alt-f6 à partir de la console graphique

A chaque nouvelle console correspond normalement une invite de login. Il est ainsi possible de se connecter plusieurs fois sous le même nom pour exécuter des programmes différents ou que plusieurs utilisateurs soient connectés en même temps sur la machine.

Commandes

Syntaxe générale d'une commande: commande [options] paramètres En général, les options sont précédées du symbole - et peuvent être groupées;

Les paramètres précisent en général les fichiers concernés (ex de commande : tar -xzvf fichier.tgz)

Les commandes sont en réalité des fichiers exécutables (ou programme, comme les .exe sous Windows). Ces fichiers exécutables peuvent se trouver à plusieurs endroits dans l'arborescence. Pour savoir où le shell va chercher les exécutables lorsqu'on tape une commande, il faut afficher le contenu de la variable $PATH. Cette variable est une variable d'environnement (elle existe aussi sous Windows). Pour afficher son contenu, il suffit de taper:

$ echo $PATH

On reçoit ensuite la liste des répertoires contenant les commandes séparés par des :.

Pages de manuel man

Les pages de manuel sont l'aide intégrée à Linux. Chaque page traite d'une commande particulière. man nom_de_commande permet d'afficher l'aide sur cette commande particulière. Cette aide est en général orientée fonctionnement et options de la commande. On peut par exemple afficher l'aide de la commande man.

$ man man

enter permet de naviguer ligne par ligne dans les pages du manuel, la barre d'espacement page par page. Certaines versions de Linux permettent également l'utilisation des flèches pour naviguer dans le texte. La touche Q permet de quitter la commande man. /mot permet de rechercher mot dans le texte (n pour passer à l'occurrence suivante, N pour la précédente).

Ne jamais hésiter à consulter le manuel relatif à chaque commande!

La commande apropos permet de faire des recherches dans les en-têtes des pages de manuel.

$ apropos ls

L'arborescence de fichiers

Un système Linux est constitué d'une arborescence de fichiers et de répertoires. Un répertoires peut contenir des fichiers et d'autres répertoires. Le répertoire au sommet de l'arborescence est le répertoire racine /. Pour désigner un fichier de façon précise et faut mentionner le chemin complet (ou absolu) dans l'arborescence en partant de la racine.

Le caractère de séparation des répertoires dans le chemin d'accès (path) est le slash / (sous DOS, c'est le backslash \)

Le chemin d'accès absolu commence toujours par la racine / (par exemple: /etc/network/interfaces).

Aucune règle n'impose l'emplacement des fichiers, mais des conventions d'utilisation se sont créées pour l'utilisation de certains répertoires.

  • / le répertoire racine
  • /bin : les fichiers exécutables (en binaire) (initialisation du système + commandes "essentielles")
  • /boot : le noyau vmlinuz et les fichiers de démarrage
  • /dev : répertoire de fichiers spéciaux, qui servent de canaux de communication avec les périphériques (disques adaptateur réseau, cartes son etc...)
  • /etc : les fichiers de configuration du système et les principaux scripts de paramétrage
  • /etc/rc.d : scripts de démarrage du système
  • /etc/X11 : scripts de configuration du serveur X (interface graphique)
  • /etc/init.d : script de contrôle des serveurs
  • /home : la racine des répertoires personnels des utilisateurs
  • /lib : les bibliothèques et les modules du noyau
  • /mnt : la racine des points de montage des périphériques. (cd, disquette, nfs, ...).
  • /opt : lieu d'installation d'applications supplémentaires (comme OpenOffice, java, ...)
  • /root : répertoire personnel du super-utilisateur root
  • /sbin : les fichiers exécutables pour l'administration du système
  • /tmp : stockage des fichiers temporaires
  • /usr : programmes accessibles à tout utilisateur; sa structure reproduit celle de la racine /
  • /var : données variables liées à la machine (fichiers d'impression, traces de connexions http, smb, ..., dans /var/log)
  • /proc : ce pseudo-répertoire contient une "image" du système

Les répertoires personnels des utilisateurs sont placés dans le répertoire /home. Chaque utilisateur y possède un répertoire à son nom. Vous pouvez créer autant de sous répertoires que nécessaire à partir de votre répertoire personnel.

Commande pwd

A tous moments, on se trouve à un endroit dans l'arborescence. Le répertoire dans lequel on se trouve s'appelle le répertoire courant (working directory). On peut afficher le répertoire courant en utilisant la commande pwd (print working directory).

Commande ls

La commande ls (list) permet de lister le contenu d'un répertoire. (Equivalent au dir du DOS). La commande ls seule liste le contenu du répertoire courant. On peut aussi lister un autre répertoire en passant le chemin de celui-ci en paramètre (exemple: ls /usr/bin).

Lorsqu'on doit passer le chemin d'un répertoire ou d'un fichier à une commande, on peut soit donner un chemin absolu (commençant par / et partant donc de la racine), soit donner un chemin relatif (ne commençant pas par / et parant du répertoire courant).

La commande ls comporte des options dont les plus utiles sont:

$ ls -a : l'option -a (all) permet d'afficher tous les fichiers, y compris les fichiers cachés. Ces fichiers cachés sont repérables au fait que leur nom commence par un point ..

$ ls -l : l'option -l (long) permet d'afficher les informations détaillées sur chaque fichier.

Exemple de résultat de la commande ls -l

- rw-r--r-- 1 dfr users 6025   Mar 15 2004  Fontmap
- rw-r--r-- 1 dfr users 139564 Dec 20 09:11 index.html
d rwxr-xr-x 2 dfr users 4096   Jan 25 07:40 bin

Les informations que l'on retrouve dans les différentes colonnes sont:

  • type de fichier: - fichier normal, d répertoire, l lien
  • permissions lecture r écriture w exécution x pour le propriétaire du fichier, le groupe d'utilisateurs et les autres
  • nombre de liens durs
  • propriétaire
  • groupe
  • taille en octets
  • date (et heure) de la dernière modification
  • nom

Les options peuvent également être combinées entre-elles: $ ls -al : liste tous les fichiers au format long.

La plupart des commandes Unix permettent l'utilisation d'options. Ces options sont en général précédées d'un tiret et doivent être séparées de la commande par un espace.

Comparez les résultats des commandes suivantes dans votre répertoire personnel:

$ ls
$ ls -l
$ ls -a
$ ls -la
$ ls -R
$ ls -rial
$ ls --help

Commande cd

La commande cd (change directory) permet de naviguer dans l'arborescence de répertoires.

Le chemin peut être précisé de manière absolue en commençant par la racine:

$ cd /etc/X11

ou de manière relative à partir du répertoire actuel:

$ pwd
/etc/X11
$ cd xdm
$ pwd
/etc/X11/xdm

La séquence .. désigne le répertoire parent, c'est à dire le répertoire immédiatement au dessus dans l'arborescence de répertoires. Pour remonter d'un répertoire dans l'arborescence:

$ cd ..

Le point seul désigne le répertoire actuel.

cd sans arguments permet de rejoindre le répertoire de travail personnel.

Auto-complétion des commandes et noms de fichiers

Les shells modernes facilitent l'édition de la ligne de commande en fournissant un outil de complètement automatique des mots.

Entrez la ligne suivante sans appuyer sur enter

$ cd /usr/inc

Pressez ensuite la touche tabulation ⇥. Le shell essaie de deviner la suite et complète le inc par lude, pour former le nom du répertoire include de /usr. Enfoncer enter validera alors la commande.

Si 2 fichiers commencent par les mêmes lettres, le shell complète les caractères communs et attend une lettre supplémentaire pour pouvoir différencier les fichiers. Bash permet d'afficher ces fichiers en poussant une seconde fois sur tabulation.

Répertoires et fichiers

Création d'un répertoire mkdir

La commande mkdir permet de créer un répertoire. Si le chemin d'accès complet n'est pas spécifié, le répertoire est créé dans le répertoire courant.

Créer le répertoire programmes dans votre répertoire de travail

$ cd
$ mkdir programmes

Ce qui revient à écrire

$ mkdir /home/student/programmes

ou (Rappel ~ représente toujours votre répertoire personnel)

$ mkdir ~/programmes

Faire de ~/programmes le répertoire courant

$ cd programmes
$ pwd
/home/ubuntu/programmes

Suppression de répertoires rmdir

La commande rmdir permet de supprimer les répertoires, à condition que ceux-ci soient vides.

$ rmdir programmes

Suppression de fichiers rm

La commande rm permet de supprimer des fichiers. Unix ne demande pas de confirmation de suppression et toute suppression est définitive!! A utiliser avec précautions, surtout lorsqu'on travaille en tant qu'utilisateur root avec tous les droits sur l'entièreté de la machine!!! L'option -r (récursive) indique à rm de supprimer le répertoire et l'intégralité de son contenu. Une commande rm -r / en tant que root efface purement et simplement le contenu de toute votre partition racine!

Copie d'un fichier cp

La commande cp (copy) permet de copier un répertoire ou un fichier. L'option -r permet d'exécuter la copie récursive d'un répertoire et de son contenu.

$ cp /etc/passwd ~/passwd.copy
$ mkdir ~/temp
$ cp passwd.copy temp/passwd.txt
$ cp -r temp temp2
$ cp -r temp temp2

Quelles sont les opérations réalisées par chacune des commandes ci-dessus? Tenir compte des 2 commandes cp -r.

Déplacement et renommage d'un fichier mv

La commande mv (move) permet de renommer un fichier ou un répertoire ou de le déplacer dans un autre répertoire.

renommer passwd.copy en passwd.old:

$ mv passwd.copy passwd.old

déplacer passwd.old dans le sous-répertoire temp existant:

$ mv passwd.old temp

Lecture du contenu d'un fichier

￼Il existe plusieurs commande permettant de lire le contenu d'un fichier. La plus simple est la commande cat (concatenate):

$ cat .bashrc

Cette commande affiche purement et simplement le contenu du fichier dans la console mais ne permet pas de naviguer dans le fichier ce qui rend impossible la consultation d'un fichier trop long pour tenir en entier sur un écran de la console.

la commande more permet d'afficher page par page le contenu du fichier. Elle permet également une navigation basique dans le fichier (enter, spacebar, B) ainsi que la recherche d'un motif (/motif). Idéale pour fichiers courts

$ more .bashrc

la commande less est encore plus complète dans ses possibilités (elle permet par exemple d'utiliser les flèches pour naviguer).

$ less .bashrc

Permissions

Pour des raisons de sécurité, il est nécessaire d'attribuer des permissions d'accès définies aux fichiers. Sécurité pour assurer la confidentialité des données mais également et surtout pour éviter les erreurs de manipulations d'un utilisateur normal.

Les permissions:

  • lecture: lire le contenu d'un fichier (r = Read)
  • écriture: modification d'un fichier (w = Write)
  • exécution: exécution en tant que programme ou bien accès à un répertoire (x = eXecute)

Les permissions sont accordées pour 3 types d'utilisateurs:

  • le propriétaire du fichier, en général celui qui a créé le fichier (u = User)
  • le groupe d'utilisateurs auquel appartient le fichier (g = Group).
  • les utilisateurs n'appartenant pas au groupe du propriétaire (o = Other).

Comme nous l'avons vu précédemment, la commande ls -l vous permet de visualiser les permissions sur les fichiers d'un répertoire

Par défaut, lors de la création d'un fichier, le propriétaire a les droits r et w, les autres n'ayant accès qu'en lecture seule.

Changement des droits d'accès

La commande chmod permet de changer les permissions d'un fichier. La commande chown permet de changer le propriétaire d'un fichier.

Le changement de groupe se fait par les commandes chown ou chgrp.

Exemple: pour ajouter les droits d'écriture à un fichier, on utilise chmod +w fichier et pour les supprimer chmod -w fichier. Les droits sont alors assignés à tous les niveaux (utilisateur, groupe, autres). Pour assiger à un niveau particulier, on précise par exemple chmod g-w fichier. Ce qui supprime le droit d'écriture aux membres du groupe seulement.

On peu aussi définir les droit sous forme octale (base 8). Les droits sont alors codés sous la forme suivante:

UtilisateurGroupeAutres
rwxrwxrwx
400200100402010421

Pour que le propriétaire soit le seul à avoir l'accès en lecture et rien d'autre: chmod 400 fichier

Pour donner à tout le monde l'accès en lecture seule: chmod 444 fichier

Pour donner les droits de lecture et écriture à tout le monde (400+200, 40+20, 4+2): chmod 666 fichier

Redirection de la sortie des programmes

Par défaut, lorsque un programme affiche quelque chose, cela s'affiche directement dans la console. Il est cependant possible de rediriger la sortie d'un programme. Par exemple, afin de pouvoir sauvegarder ou lire à son aise la sortie d'un programme, il est utile de pouvoir rediriger la sortie vers un fichier.

Le signe > suivi d'un nom de fichier permet de rediriger la sortie du programme vers ce fichier. Attention, si le fichier existait déjà, il est écrasé par la nouvelle sortie du programme.

$ ls /usr/bin > ~/listbin.txt

listbin.txt contient le résultat de la commande ls, soit le contenu du répertoire /usr/bin.

Pour insérer à la suite du contenu existant au lieu d'écraser le contenu du fichier, le signe >> peut être utilisé.

$ date >> ~/listbin.txt

Il existe 2 flux de sortie pour la plupart des programmes Unix. L'un est la sortie standard (stdout), l'autre la sortie d'erreurs (stderr). Le signe > ne capture que la sortie standard sans les erreurs. Le signe &> permet de rediriger les 2 sorties.

$ gcc invinitjig.c &> erreurs.log

La sortie standard reçoit le numéro 1 et les erreurs le numéro 2, ce qui permet éventuellement de les traiter séparément.

$ gcc invinitjig.c 2> erreurs.log 1> output.log

En utilisant la commande cat avec une redirection, il est possible de faire une édition minimale d'un fichier:

cd : pour aller dans votre répertoire personnel

touch essai.txt : pour créer un fichier vide

cat essai.txt : pour vérifier que le fichier est vide

cat > essai.txt : la commande cat sans fichier en paramètre attend que l'on tape quelque chose et le recopiera en sortie (et donc dans essai.txt). On utilise Ctrl-d pour terminer la saisie.

Je suis heureux d'apprendre à travailler avec LINUX : écrit cette phrase dans le fichier.

Ctrl-d : pour terminer et enregistrer

cat >> essai.txt : pour ajouter du texte à la suite

système d'exploitation multi-utilisateurs et multi-tâches : ajoute cette phrase à la suite du fichier.

Ctrl-d

cat essai.txt : pour afficher le contenu du fichier

Il est également possible de rediriger la sortie d'un programme vers un autre programme. Par exemple, si la sortie d'un programme est trop longue pour tenir sur un écran, il peut être intéressant de la rediriger vers le programme less. Pour cela, on utilise le caractère "pipe" |.

$ ls /bin | less

Expansion

Que fait la touche Enter?

Nous allons explorer ici ce qu'il arrive à nos commande lorsqu'on les valide avec la touche enter.

Pour commencer nous allons utiliser la commande echo qui affiche du texte.

$ echo hello world !!
hello world !!

Si nous essayons maintenant la commande suivante:

$ echo *
Desktop Documents ls-output.txt Music Pictures Public Templates Videos

Le résultat peut varié an fonction du contenu du répertoire courant. Que s'est-il passé? Pourquoi l'astérisque ne s'est pas affichée? Le mécanisme à l'oeuvre ici est appelé "expansion".

Les commandes subissent une expansion avant d'être exécutée. Dans ce cas-ci l'* est remplacée par les noms des fichiers du répertoire courant. La commande echo affiche ensuite ces noms.

Expansion des noms de fichiers

L'expansion de l'* est gérée par le mécanisme d'expansion des noms de fichiers. l'* fait partie d'un ensemble de notations appelé "wildcards" et permettant de référencer plusieurs noms de fichiers en une fois. Voici les autres "wildcards"

  • *: représente zéro, un ou plusieurs caractère quelconques.
  • ?: représente un caractère quelconque.
  • [caractères]: représente n'importe quel caractère se trouvant entre les []. Le - sert à définir une plage de caractères.
  • [!caractères]: représente n'importe quel caractère ne se trouvant pas entre les []. Le - sert à définir une plage de caractères.

Les [] et les [!] peuvent également contenir des classes prédéfinies de caractères. Ces classes ont la forme suivante: [:class:]. Voici les classes qui existent:

  • [:alnum:]: Les caractères alphanumériques
  • [:alpha:]: Les caractères alphabétiques
  • [:digit:]: Les caractères numériques
  • [:lower:]: Les caractères en minuscules
  • [:upper:]: Les caractères en majuscules

Exemples:

$ echo D*
Desktop Documents

$ echo *s
Documents Pictures Templates Videos

$ echo [[:upper:]]*
Desktop Documents Music Pictures Public Templates Videos

Expansion du ~

Le ~ est transformé lorsqu'il est au début d'un nom d'utilisateur en le chemin vers le répertoire personnel de l'utilisateur.

~foo devient /home/foo

~ devient /home/student. Si aucun nom n'est spécifié, c'est l'utilisateur courant qui est utilisé.

Expansion arithmétique

Un certain nombre de calcul peuvent être réalisé par expansion en utilisant la notation $(()).

$ echo $((2+2))
4

L'expansion arithmétique ne fonctionne qu'avec les entiers. Voici une liste non-exhaustive des opération supportées:

  • +: somme
  • -: différence
  • *: multiplication
  • /: division entière
  • %: modulo
  • **: exponentiation

On peut utilisé des parenthèses pour expliciter l'ordre des opérations

Expansion des accolades

Cette expansion permet de créer plusieurs chaines de caractères à partir d'un modèle contenant des accolades:

$ echo Front-{A,B,C}-Back
Front-A-Back Front-B-Back Front-C-Back

Les accolades peuvent contenir une liste de chaines de caractères séparée par des virgules ou un intervalle de caractères:

$ echo Number_{1..5}
Number_1 Number_2 Number_3 Number_4 Number_5

$ echo {Z..A}
Z Y X W V U T S R Q P O N M L K J I H G F E D C B A

Ces expansions peuvent être imbriquées:

$ echo a{A{1,2},B{3,4}}b
aA1b aA2b aB3b aB4b

Expansion des variables

Dans un environnement Linux, il existe un certain nombre de variable. Pour que les nom de variables soient remplacé par leur contenu, on les fait précéder d'un $.

$ echo $USER
student

Vous pouvez afficher la liste des variables avec printenv.

Substitution de commande

les $() nous permettent d'utiliser la sortie d'une commande comme une expansion.

$ echo $(ls)
Desktop Documents ls-output.txt Music Pictures Public Templates Videos

Le résultat de la commande ls vient prendre la place de $(ls) et echo affiche le résultat.

Contrôler l'expansion

l'expansion est très utile mais dans certain cas, il faut pouvoir la contrôler. Voici quelques exemple où l'expansion peut être non désirée:

$ echo This is a      test
This is a test
$ echo The total is $100.00
The total is 00.00

Dans le premier exemple, chaque mot est considéré comme un paramètre de la commande echo indépendamment du nombre d'espaces (qui ne sont considérés que comme des délimiteurs séparant les paramètres). Dans le deuxième, la variable $1 est considérée mais elle n'est pas définie et est donc remplacée par une chaine vide.

Les guillemets

Entre des guillemets, les caractères spéciaux perdent leur statuts particuliers à l'exception du $. Cela signifie que la séparation des paramètres, l'expansion des nom de fichier, l'expansion des tilde et l'expansion des accolades ne fonctionnent pas dans les guillemets. Par contre, l'expansion des variables, l'expansion arithmétique et la substitution de commande fonctionnent toujours dans les guillemets.

Les guillemets peuvent, par exemple, servir pour les noms de fichier contenant des espaces.

Les apostrophes

Pour supprimer toutes formes d'expansion, on utilise les '.

Introduction à vi

vi est un éditeur de texte. Il vous permettra donc d'éditer des scripts, des fichiers de configuration et des codes sources. Cet éditeur n'est pas connu pour être facile à utiliser, bien au contraire. Pour commencer à être à l'aise avec vi, il faut de la pratique et pour être bon, il faut beaucoup de pratique.

Mais pourquoi faut-il apprendre à utiliser vi s'il est si difficile à utiliser? Voici déjà quelques bonnes raisons:

  • vi est installé sur tous les systèmes UNIX. Le standard POSIX qui définit les systèmes UNIX exige qu'il soit présent.
  • vi est léger et rapide. Il démarre rapidement et permet d'éditer un fichier extrêmement rapidement.

Pour démarrer vi, il suffit de taper la commande vi.

$ vi

Pour quitter vi, il suffit de taper la commande :q. Si on est dans un fichier qui contient des changements non sauvegardés, vi refusera de quitter. Dans ce cas, si on souhaite vraiment quitter en abandonnant les modifications non sauvées, il suffit d'entrer la commande :q!.

Si pour ouvrir vi on ajoute un nom de fichier à la commande, cela ouvre le fichier en question ou cela le crée s'il n'existait pas encore.

$ vi foo.txt

vi possède plusieurs modes. Lorsqu’il démarre, il est en mode commande. Dans ce mode, presque chaque touche correspond à une commande. Il faut donc être prudent lorsqu'on est dans ce mode au risque de faire de grosses bêtises.

Pour commencer à écrire dans le fichier, il faut passer en mode insertion avec la touche i. Lorsqu'on est en mode insertion -- INSERT -- s'affiche en bas de la console. Pour quitter le mode insertion, utilisez la touche Esc.

Pour sauver les modifications, il faut utiliser la commande :w lorsqu'on est en mode commande. Dans vi, il y a deux types de commandes:

  • Celles qui commencent par :. Elles s'affichent en bas de la console lorsqu'on les écrit et il faut les valider avec Enter.
  • Celles qui ne commencent pas par :. Elles ne s'affichent pas et s'exécutent dès que le dernier caractère de la commande est tapé.

Voyons maintenant comment déplacer le curseur en mode commande. Il existe plusieurs commandes permettant d'obtenir toutes sortes de déplacements:

j (ou la flèche bas)une ligne vers le bas
k (ou la flèche haut)une ligne vers le haut
0au début de la ligne

wau début du mot suivant

Ctrl-F (ou Page Up)à la page suivante

number-Shift-Gà la ligne number
G (Shift-g)à la dernière ligne du fichier

Beaucoup de commandes peuvent être préfixées d'un nombre permettant d'indiquer le nombre de fois que la commande doit être répétée. Par exemple, 5j déplace le curseur de 5 lignes vers le bas. Cependant, le nombre n'indique pas toujours un nombre de répétition. Le cas de la commande number-Shift-G, décrite plus haut, en est un exemple.

Nous allons maintenant commencer à voir comment éditer du texte avec vi. La première chose à connaitre, c'est la manière de faire un Undo. Pour annuler un changement, il suffit de taper la touche u en mode commande.

Nous avons déjà vu comment passer en mode insertion avec la touche i. Si vous vous trouvez sur le dernier caractère d'une ligne et que vous passez en mode insertion, le texte que vous taperez ensuite se trouvera avant le dernier caractère de la ligne. Il existe donc une commande sur la touche a permettant elle d'ajouter du texte après le caractère courant. Comme il est fréquent de vouloir ajouter après le dernier caractère de la ligne, il y a aussi une commande pour ça, A.

Il est également fréquent de vouloir ajouter du texte dans une nouvelle ligne. Il existe donc deux commandes pour ça. o permet d'ajouter du texte dans une nouvelle ligne située en dessous de la ligne courante et O permet de faire la même chose au dessus.

Maintenant que nous savons ajouter du texte, nous allons voir comment en supprimer. Commençons par la commande x qui permet de supprimer un caractère. Elle peut être préfixée d'un nombre pour supprimer plusieurs caractères. Le commande x est assez simple. La commande d est plus subtile et plus puissante. Cette commande utilise un mouvement pour spécifier l'ampleur de la suppression. Voici quelques exemple:

CommandeSuppression
dwde l'emplacement courant jusqu'au début du mot suivant
d$de l'emplacement courant jusqu'à la fin de la ligne
d0de l'emplacement courant jusqu'au début de la ligne
dGde la ligne courante jusqu'à la fin du fichier
d20Gde la ligne courante jusqu'à la ligne 20
ddde la ligne courante
3ddde la ligne courante plus les deux qui suivent

Une des choses très utiles lorsqu'on édite du texte est le copier-coller. Il est tout à fait possible de faire des copier-coller ou couper-coller dans vi. Lors d'une suppression avec la commande d, cela fait en réalité un "couper". Pour coller ce qui vient d'être coupé après le curseur, on utilise la commande p. On peut également coller avant le curseur avec P.

Pour copier du texte sans le supprimer, on utilise la commande y qui s'utilise de la même façon que d.

CommandeCopie
ywde l'emplacement courant jusqu'au début du mot suivant
y$de l'emplacement courant jusqu'à la fin de la ligne
y0de l'emplacement courant jusqu'au début de la ligne
yGde la ligne courante jusqu'à la fin du fichier
y20Gde la ligne courante jusqu'à la ligne 20
yyde la ligne courante
3yyde la ligne courante plus les deux qui suivent

Il est également très utile de pouvoir chercher dans le fichier qu'on édite. Pour cela, on procède exactement de la même façon que dans less. /mot pour chercher mot. n pour passer à l'occurrence suivante et N pour passer à la précédente.

On peut aussi faire un search & replace. Imaginons que l'on veuille remplacer pattern par replacement dans tout le fichier, la commande à utiliser est :%s/search/replacement/g.

Examinons les différentes parties de cette commande:

PartieDescription
:Début de commande
%Indique que l'opération doit se faire sur tout le fichier. Il est aussi possible de spécifier un intervalle de ligne. Par exemple, 2,5 indique d’effectuer l'opération de la ligne 2 à la ligne 5. 1,$ indique de la première à la dernière ligne (comme %). Si on omet de spécifier l'intervalle, l'opération ne se fait que sur la ligne courante.
sL'opération à effectuer. s pour substitution (search & replace).
/search/replacement/Le string à chercher et le string de remplacement.
gSpécifie que l'opération est globale et que toutes les occurrences seront remplacées. Sans le g, seule la première occurrence serait remplacée.

Pour finir, nous allons voir comment ouvrir plusieurs fichiers en même temps. Cela peut servir pour copier du texte d'un fichier à l'autre.

On peut ouvrir plusieurs fichiers au moment du lancement de vi en indiquant plusieurs noms de fichiers:

$ vi file1 file2 file3 ...

On peut aussi ouvrir un fichier supplémentaire lorsqu'on est déjà dans vi avec la commande :e nom_de_fichier.

Une fois que plusieurs fichiers sont ouverts, on peut passer de l'un à l'autre avec les commandes :n et :N pour passer au fichier suivant ou précédent.

De plus, la commande :buffers permet d'afficher la liste des fichiers ouverts. Dans cette liste, chaque fichier ouvert possède un numéro. Pour passer directement à un fichier, il suffit d'utiliser la commande :buffer numéro_de_fichier.

Recherche de fichier avec find

la commande find permet de chercher des fichiers selon différents critères. Ces critères sont contrôlés par les options suivantes:

OptionDescriptionExemple
-nameRecherche par nom de fichier.find ~ -name "*.pdf"
-typeRecherche par type de fichier. Exemple: d pour répertoire, f pour fichier, ...find . -type d
-userRecherche par nom de propriétaire.find /home -user student
-groupRecherche par nom de groupe.find /bin -group root
-sizeRecherche par taille de fichier.find ~ -size -100c
-atimeRecherche par date de dernier accès.find . -atime -3
-mtimeRecherche par date de dernière modification.find . -mtime +3
-ctimeRecherche par date de création.find . -ctime -3
-permRecherche par permission.find . -perm 755
-aPermet de faire un "et" entre plusieurs critères. Cette option peut être omise puisque le "et" est fait d'office lorsqu'on met plusieurs critère à la suite.find . \( -name '*.tex' -a -name '*.dvi' \)
-oPermet de faire un "ou" entre plusieurs critères.find . \( -name '*.tex' -o -name '*.dvi' \)
!Permet de nier un critère.find / ! -user root

Pour les options -size, -atime, -mtime et -ctime, le signe + permet de dire "supérieur à", - permet de dire "inférieur à", et rien permet de dire "égal à".

L'option -exec permet d'exécuter une commande sur les fichiers trouvés: find . -type f -exec ls -l {} \; cherche tous les fichiers du répertoire courant et execute la commande ls -l sur chacun d'entre eux. Le {} dans la commande à executer représente le fichier trouvé. Le \; marque la fin de la commande permettant ainsi de mettre d'autres options pour la commande find à la suite.

Expressions régulières

Comme nous l'avons vu, dans un système Linux, les données de type texte sont très importante. Il est donc très importants d'avoir des outils puissants pour manipuler le texte et parmi ces outils, un des plus puissants est l'utilisation d'expressions régulières.

Les expressions régulières sont des notations permettant d'identifier des pattern de texte. Il est important de garder à l'esprit que beaucoup de commandes et de langages supportent les expressions régulière. Malheureusement, il arrive souvent qu'elles varient légèrement d'un outil à l'autre. Nous nous intéresserons ici aux expressions régulières définies dans le standard POSIX.

La commande que nous allons utiliser pour utiliser des expressions régulières est grep (global regular expression print). Cette commande cherche dans des fichiers texte et affiche les lignes contenant une occurrence d'une expression régulière. Il est courant d'utiliser grep avec un pipe pour chercher dans l'output d'une autre commande.

$ ps aux | grep init

Dans cet exemple, nous utilisons grep pour chercher une chaine fixe. C'est la forme la plus simple d'expression régulière.

La forme générale de la commande grep est la suivante: grep [options regex [file...]] où regex est une expression régulière.

Voici quelques options de grep:

OptionDescription
-iignore la case
-vaffiche les lignes qui ne contiennent pas de correspondance avec l'expression régulière.
-caffiche le nombre de correspondances plutôt que les lignes contenant une correspondance.
-laffiche le nom des fichiers qui contiennent une correspondance.
-Laffiche le nom des fichiers qui ne contiennent pas de correspondance.
-naffiche les numéros des lignes affichées

Une expression régulière est constituée de caractères. Parmi eux, certains sont des caractères littéraux et d'autres sont des méta-caractères. Les caractères littéraux sont cherchés tels quels alors que les méta-caractères servent à chercher des motifs plus complexes. Voici la liste des méta-caractères: ^ $ . [ ] { } - ? * + ( ) | \. Tous les autres caractères sont des littéraux. Pour chercher une chaine qui contient un des méta-caractères, il faut le faire précéder d'un \.

Il est important ici de constater que plusieurs des méta-caractères servent déjà au processus d’expansion de commande. Comme l’expansion a lieu avant l’exécution de la commande, elle pourrait modifier l'expression régulière avant qu'elle ne soit utilisée par la commande grep. Pour cette raison, il est souvent indispensable de protéger l'expression régulière de l’expansion avec des '.

Méta-caractèreDescription
.Représente un caractère quelconque.
^Représente le début d'une ligne. ^Salut correspond à "Salut" écrit en début de ligne.
$Représente la fin d'une ligne. ^$ correspond à une ligne qui commence et se finit juste après, c'est à dire, une ligne vide.
[]Permet de créer une classe de caractères. Une classe correspond à un caractère dans la chaîne à trouver. On liste les caractères autorisés entre les crochets. Par exemple, l'expression gr[oa]sse trouvera à la fois le mot "grosse" et le mot "grasse". Les méta-caractères peuvent être mis tels quels dans une classe à l'exception du ^ et du - qui ont un sens particulier.
  • le ^ en début de classe sert à indiquer la négation. La classe correspond alors aux caractères qui ne sont pas dans la liste. Si on veut une classe qui permette de trouver un ^, il suffit de ne pas le mettre en premier dans la liste.
  • le - sert à indiquer un intervalle de caractères. [A-Z] correspond à toutes les lettres majuscules. Une classe peut contenir des caractères individuels et des intervalles ([a-z0-9+] est une classe valide). Si on veut une classe qui permette de trouver un -, il suffit de le mettre en premier dans la liste. Attention, les intervalles dépendent de l'ordre considéré pour les caractères et cet ordre peut varier d'un système à l'autre. Dans certains systèmes, les lettres sont rangées comme ceci ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz et dans d'autres comme ceci aAbBcCdDeEfFgGhHiIjJkKlLmMnNoOpPqQrRsStTuUvVwWxXyYzZ. Pour cette raisons il existe des classes toutes faites:
    • [:alnum:]: Les caractères alphanumériques
    • [:alpha:]: Les caractères alphabétiques
    • [:digit:]: Les caractères numériques
    • [:lower:]: Les caractères en minuscules
    • [:upper:]: Les caractères en majuscules
    • [:punct:]: Les caractères de ponctuation
    • [:blank:]: L'espace et le TAB
    • [:space:]: L'espace, le TAB, le retour à la ligne et quelques autres caractères moins courants.

    Exemple: [[:lower:][:digit:]+] est une classe qui accepte les minuscules, les chiffres et le caractère +.

Comme si les choses n'étaient pas déjà assez compliquée, POSIX divise le standard des expressions régulières en deux: les expressions régulières de base (BRE) et les expressions régulières étendues (ERE). En BRE, les seuls méta-caractères reconnus sont ^ $ . [ ] * alors qu'en ERE, ils sont tous reconnus. Le plus marrant, c'est qu'il est quand même possible d'utiliser les {} et les () en BRE en les précédant d'un \. Par défaut, grep utilise les BRE. Il faut lui préciser l'option -E pour qu'il utilise les ERE ou utiliser la commande egrep.

Méta-caractèreDescription
|Ce caractère permet de réaliser des "OU". L'expression régulière lundi|mardi|mercredi|jeudi|vendredi|samedi|dimanche permettra de trouver les jours de la semaine. Il est peut être nécessaire d'utiliser des () pour séparer un "OU" du reste de l'expression. Exemple: (in|out)put
?Il s'agit du premier quantificateur que l'on rencontre. Il permet d'indiquer que l'élément qui précède doit apparaitre zéro ou une fois. En bref, il indique que l'élément qui précède est optionnel. Exemple: gâteaux? trouvera "gâteau" et "gâteaux". On peut bien sûr utiliser des () pour appliquer ce quantificateur à tout une expression régulière. Exemple: tartes? (à la crème)?.
*Il s'agit ici du seul quantificateur disponible directement en BRE. Il indique que l'élément qui précède peut apparaitre zéro, une ou plusieurs fois. On peut aussi utiliser des () pour appliquer ce quantificateur à tout une expression régulière
+Ce quantificateur ci indique que l'élément qui précède peut apparaitre une ou plusieurs fois.
{}Les {} servent aussi de quantificateur et permettent d'indiquer précisément le nombre d’occurrences pouvant apparaitre dans la chaine à chercher.
  • {n} l'élément qui précède peut apparaitre n fois.
  • {m,n} l'élément qui précède peut apparaitre entre m et n fois.
  • {m,} l'élément qui précède peut apparaitre m fois ou plus.
  • {,n} l'élément qui précède ne peut apparaitre plus de n fois.

Les {} sont accessibles en BRE à condition de les faire précéder d'un \.

Il peut être intéressant de noter que vous pouvez utiliser les expressions régulières lorsque vous faites une recherche avec / dans less ou dans vi.
