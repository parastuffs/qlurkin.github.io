Projet d'application mobile

Énoncé

On vous demande de créer l'application mobile correspondant à l'API que vous développez pour le cours d'Architecture Web.

Modalités

Les groupes sont les même que pour le cours d'Architecture Web.

L'application doit être native. Vous pouvez choisir librement les outils de développement.

Votre code doit se trouver sur un git accessible par Quentin Lurkin.

Vous devrez présenter ce projet lors de l'examen oral en session.

Grille d'évaluation

  • L'application démarre (1 points)
  • L'application utilise l'API du projet d'Architecture Web. Elle en respecte les contraintes fonctionnelles: (4 points)
    • Pouvoir lister des objets
    • Afficher un objet précis et les liens avec d’autres objets
    • Rechercher un objet
    • Ajouter un objet
    • Modifier un objet
    • Supprimer un objet
    • Lier un objet à un autre objet
    • Authentifier l’utilisateur
  • L'état de l'UI est préservé à la suite d'un redémarrage de l'activité. (4 points)
  • Aucune opération ne bloque l'UI. (3 points)
  • La présentation décrit le contexte du projet. (1 points)
  • La présentation justifie les choix graphiques (design) sur base des notions vues au cours. (4 points)
  • Le code sur git est clair et documenté. L'historique est significatif. (3 points)
