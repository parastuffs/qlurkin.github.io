Boutons avec Icone

Ce genre de button est inaccessible aux Screen Reader

<button><img src="img/icon.svg"/></button>

Solution:

  • L'attribut aria-label permet de définir un label à destination des Screen Reader 
    <button aria-label="Search"><img aria-hidden="true" src="img/search.svg"/></button>

  • On peut aussi ajouter un texte caché dans le bouton

    <button>
	<img src="img/search.svg" aria-hidden="true"/>
	<span class="accessible-hidden"> Search </span> 
</button>
    span.accessible-hidden {
	position: absolute;
	left: -1000r0px;
}

    Remarquez que l'on utilise pas l'attribut CSS display: none car cela cache le contenu pour les Screen Reader également

  • On peut, en bonus, fournir un tooltip au survol de la souris 

    button {
	position: relative;
}

span.accessible-hidden {
	position: absolute;
	left: -1000r0px;
}

button:hover span.accessible-hidden {
	left: 50%;
	margin-left: -50%;
	top: -2.5em;
	display: block;
	color: white;
	background-color: rgba(0,0,0,0.6);
	font-size: 0.6em;
	padding: 0.5em;
	border-radius: 5px;
}

Structure

  • Utiliser les bonnes balises pour chaque élément de structure de la page

    Beaucoup de balises HTML ont une valeur sémantique. Par exemple, les balises de titre (<h1>, <h2>, ...) permettent de créer une structure hiérarchique à un document.

    Il en existe beaucoup d'autres: <header>, <footer>, <main>, <nav>, <aside>, ...

  • Utiliser les aria role: search, banner, navigation, ... 
    <div role="navigation"></div>
<nav></nav>

    Les aria role permettent d'attribuer un rôle sémantique à une balise qui n'en a pas ou d'écraser le rôle par défaut d'une balise qui en déjà un.

    Attention qu'une mauvais utilisation des aria role peut dégrader l'accessibilité au lieu de l'améliorer:

    Mauvais exemple:

    
            <div id="nav">
            <ul role="navigation">
            ...
            </ul>
            </div>

    Le rôle de liste de la balise <ul> a été remplacé par un rôle de navigation.

    Bon exemple:

    
            <div role="navigation">
            <ul>
            ...
            </ul>
            </div>

    Les rôles de navigation et de liste sont préservés

    Le mieux reste d'utiliser les balises dédiées lorsqu'elles sont disponibles.

    
            <nav>
            <ul>
            ...
            </ul>
            </nav>

Définir la langue

Pour que les Screen Reader puisse utiliser l'algorithme de synthèse vocale adapté, il est important de spécifier la langue du contenu.

<html lang="fr">
	...
	<p> Définir le <em lang="en">layout</em> du site.</p>
	...
</html>

Navigation sans souris

L'utilisation de la souris n'est pas possible pour tout le monde. Il faut que la navigation au clavier soit possible et quelle soit aussi pratique que possible.

  • Definir un ordre de navigation par tabulation logique 
    <a href="https://www.w3schools.com/" tabindex="2">W3Schools</a>
<a href="http://www.google.com/" tabindex="1">Google</a>
<a href="http://www.microsoft.com/" tabindex="3">Microsoft</a>
  • Ajouter un lien vers le contenu principal 
    <body>
	<a href="#main">Skip to main content</a>
	<nav role='navigation'>
		<a href="/">Home</a>
		<a href="/about">About</a>
		<a href="/blog">Blog</a>
	</nav>
	<main id="main" role="main">
	<!-- page specific content -->
	</main>
</body>

Formulaires

  • Définir le l'ordre de navigation par tabulation
  • Ajouter des <label> à chaque élément du formulaire 
    <label for="name">Name:</label>
<input id="name" type="text">

    <label> Name:
	<input type="text">

Moteur de recherche

Meta-data

  • Le titre est important 
    <title>Un super titre</title>
  • Ajouter une description. Elle apparait directement dans les résultats de recherche 
    <meta name="description" content="description du site"/>
  • Meta robots 
    <meta name="robots" content="noindex, nofollow">

    noindex: le moteur de recherche n'indexera pas la page

    nofollow: le moteur de recherche ne suivra pas les liens de la page. Il peut être intéressant d'utiliser cet attribut sur les pages dont on ne contrôle pas le contenu (contenu produit par les utilisateurs).

  • Ajouter des données structurées à votre contenu.

    Ces données structurées remettent aux moteurs de recherche de présenter des résultats plus riches :

Données stucturée: Microdata

https://schema.org

<div itemscope itemtype="http://schema.org/Movie">
	<h1 itemprop="name">Avatar</h1>
	<div itemprop="director" itemscope itemtype="http://schema.org/Person">
		Director: <span itemprop="name">James Cameron</span> (born <span itemprop="birthDate">August 16, 1954</span>)
	</div>
	<span itemprop="genre">Science fiction</span>
	<a href="../movies/avatar-theatrical-trailer.html" itemprop="trailer">Trailer</a>
</div>

https://json-ld.org/
